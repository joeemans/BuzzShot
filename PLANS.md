# BuzzShot V2 Plan

The canonical product plan is `.agents/PLANS.md`. This root file exists so the project follows the target repository structure and so contributors know where to start.

Implementation follows the same milestones:

1. Clean monorepo bootstrap.
2. Local PostgreSQL and Redis infrastructure.
3. NestJS API foundation.
4. Prisma schema and seed data.
5. Authentication.
6. TMDB-backed discovery and detail pages.
7. Ratings, reviews, watch states, lists, follows, feed, and recommendations.
8. Production hardening, Docker, QA, and documentation.

Any change from that plan must be recorded in `CHANGES.md`.
